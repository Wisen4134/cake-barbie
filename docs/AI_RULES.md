# AI Agent Rules

## 🎯 目標
協助開發專案，但不能取代開發者思考與決策。

---

## 📌 行為原則

### 1. 分段實作（重要）
- 不可一次生成整個專案
- 每次只實作一個功能（例如 Login API）

---

### 2. 不可擅自擴展範圍
- 只根據 PROJECT_SPEC.md 實作
- 不可自行新增未定義功能

---

### 3. 程式碼需可解釋
- 所有產出的 code 必須可以說明用途
- 若使用較進階寫法，需附上簡單解釋

---

### 4. 安全限制（非常重要）
- 不可存取工作區以外的檔案
- 不可讀取或修改系統敏感資料
- 不可下載或執行未知來源程式

---

### 5. 套件與資源
- 優先使用官方或常見套件
- 不可隨意引用不明來源 GitHub 專案
- 若使用套件，需說明用途

---

### 6. 開發方式
- 優先使用簡單、可理解的實作
- 避免過度設計（Over Engineering）

---

### 7. 回覆方式
- 先說明做什麼
- 再提供 code
- 必要時提供測試方式

---

## ❗ 禁止事項

- 一次生成整個系統
- 引入過多不必要技術
- 未經允許修改專案結構
- 使用過於複雜或難以理解的寫法
